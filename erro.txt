 Tarefa sobre Integração Contínua

Tendo como base o assunto desenvolvido nos demais trabalhos (APIs, Front, Docker), agora você criará pipelines para disponibilizar os serviços em containers Docker de forma automatizada com a ferramenta Jenkins.

 

Criar um ambiente CI/CD para fazer a integração e entrega contínua dos serviços (publicação).

Exemplo de arquitetura de tecnologias envolvidas no Jenkins:






Crie um Job para cada projeto que automatize o processo de implantação das aplicações contidas nos repositórios Git.

 

O que o Job deverá fazer:

Baixar o conteúdo do repositório Git;

Fazer o build da imagem descrita no arquivo Dockerfile;

Criar um container com a imagem gerada;

Inicialize o container do Docker.

 

Observação: o job deve remover o container e a imagem existente da aplicação web antes de fazer o build da nova imagem e a criação do novo container da aplicação web.

 

Configure um gatilho para executar este processo todos os dias em um horário estabelecido.

 

Configure o envio de e-mail: ao final da execução de cada job, enviar um e-mail para você avisando sobre o sucesso ou falha na execução do job. Configure um servidor SMTP ou IMAP para enviar os e-mails (do gmail, outlook, hotmail, etc).
