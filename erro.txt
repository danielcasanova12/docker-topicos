Resumindo
Instale o CLI no próprio container Jenkins (opção 1) ou use agent docker { image 'docker:xx' args '-v /var/run/docker.sock:/var/run/docker.sock' } (opção 2).

Monte o socket Docker (-v /var/run/docker.sock:/var/run/docker.sock) no docker run do Jenkins ou no seu nó de build.

Corrija o mail no post do seu pipeline, declarando to: 'seu@email'.

Com isso o docker ... deixará de “não ser encontrado” e os seus jobs poderão buildar, destruir e iniciar containers normalmente.
